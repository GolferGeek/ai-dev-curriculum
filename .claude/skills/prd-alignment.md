# PRD alignment

When this skill applies:

- Every **PRD** goal should trace to a line in the **intention** (why / who / good).
- **Non-goals** must be explicit so scope doesn’t creep during `/run-plan`.
- **Plans** must justify the PRD: milestones map to requirements; risks name what could break verification.
- Call out **drift**: if the plan introduces features not in the PRD, stop and update PRD or cut scope.
